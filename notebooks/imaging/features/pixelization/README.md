The `pixelization` folder is the canonical PyAutoGalaxy example for modelling a galaxy whose light has two very
different components:

- A smooth central **bulge**, fit by a parametric `Sersic` light profile.
- **Asymmetric, clumpy star formation**, reconstructed on a pixel-grid (a pixelization) regularized so the
  reconstruction stays smooth.

The dataset (`dataset/imaging/clumpy`) is generated by `simulator.py` and is referenced by every other script in
this folder. Parametric profiles capture the bulge cleanly but leave structured residuals where the clumps live;
the pixelization absorbs whatever the Sersic cannot describe on a flexible pixel grid.

# Files

- `simulator`: Simulates the `clumpy` galaxy dataset — central Sersic bulge plus several off-centre Gaussian clumps.
- `modeling`: Fits the bulge + pixelization model via a non-linear search.
- `fit`: Performs a single hand-tuned bulge + pixelization fit and inspects the inversion internals.
- `likelihood_function`: A step-by-step guide of the pixelization likelihood function on the clumpy dataset.
- `galaxy_reconstruction`: How to export the pixelized galaxy reconstruction to a .csv file which can be loaded to analyse the reconstruction without PyAutoGalaxy.

# Results

These scripts only give a brief overview of how to analyse and interpret the results of a pixelization fit.

A full guide to result analysis is given at `autogalaxy_workspace/*/guides/results`.
