The `pixelization` folder is the PyAutoGalaxy example for modelling interferometer visibilities of a galaxy with
complex, irregular morphology using a pixelization.

The dataset (`dataset/interferometer/clumpy`) is generated by `simulator.py` — the same bulge + clumpy galaxy used
by the imaging tutorial in `scripts/imaging/features/pixelization`, Fourier transformed to the uv-plane. On
interferometer data the pixelization absorbs both the smooth bulge and the irregular clumps cleanly, so the
modeling example reconstructs the entire galaxy on the mesh (the imaging counterpart pairs a parametric `Sersic`
bulge with the pixelization).

# Files

- `simulator`: Simulates the `clumpy` interferometer dataset.
- `modeling`: Galaxy modeling using a pixelized reconstruction of the clumpy galaxy.
- `fit`: Performs a single hand-tuned pixelization fit and inspects the inversion internals.
- `likelihood_function`: A step-by-step guide of the pixelization likelihood function on the clumpy dataset.
- `galaxy_reconstruction`: How to export the pixelized galaxy reconstruction to a .csv file which can be loaded to analyse the reconstruction without PyAutoGalaxy.
- `many_visibilities_preparation`: How to prepare the linear algebra for datasets with many visibilities for efficient memory usage and run times.

# Results

These scripts only give a brief overview of how to analyse and interpret the results of a pixelization fit.

A full guide to result analysis is given at `autogalaxy_workspace/*/guides/results`.
